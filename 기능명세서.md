7. 관리자 인증 보안 [완료]
----------------------------------------------------------------

[인증 방식]
- ID + 비밀번호 (Google 계정 불사용)
- 비밀번호 저장 : pgcrypto bcrypt 해싱 (crypt + gen_salt('bf'))
- 비교 방식 : crypt(입력값, stored_hash) = stored_hash (복호화 불가)

[로그인 실패 정책]
- 5회 연속 실패 → admin_status = 'locked', 30분 잠금
- 자동 해제 : admin_locked_until 경과 시
- 수동 해제 : 슈퍼관리자 DB 직접 처리 (UI 예정)
- 로그인 성공 시 fail_count 초기화

[세션 관리]
- 방식 : 브라우저 sessionStorage (탭/창 닫으면 자동 소멸)
- 저장 항목 : adm_id, adm_name, adm_super
- 서버 측 세션 없음 (GAS CacheService 미사용)

[보안 설정]
- Supabase RLS : admin_master, user_textnotice 모두 활성화
- anon key : 브라우저 노출 (설계 의도) — RLS + SECURITY DEFINER로 보호
- service_role key : GAS(서버)에서만 사용, 브라우저 미노출

11. Google Sheets 연동 [완료]
----------------------------------------------------------------

- 담당 파일 : Code.gs (GAS, Sheets 컨테이너 바운드)
- 동작 방식 :
  1. 관리자가 Google Sheets '공지사항' 시트에 내용 입력
  2. A열(체크박스) 체크 → onSheetEdit 트리거 자동 실행
  3. 신규 행 → Supabase user_textnotice INSERT
     기존 행 (B열에 id 있음) → 변경 감지 후 PATCH
  4. 처리 완료 → 체크박스 해제, 성공/실패 toast 표시

- Sheets 컬럼 구성 :
  A: 체크박스(전송 트리거) / B: textnotice_id(자동기록) /
  C: 일시 / D: 작성자 / E: 제목 / F: 내용 / G: 읽음 / H: 게시여부

[별첨] 외부 접속 화이트리스트 — 방화벽/프록시 허용 도메인
================================================================
방화벽·프록시가 외부 접속을 화이트리스트(기본 차단) 방식으로
관리할 경우, 사이트 정상 동작을 위해 아래 도메인을 허용해야 함.
기준 : 현재 배포된 index.html / admin.html 코드에서 호출하는
       모든 외부 리소스 (전수 추출).

[A. 브라우저(이용자 PC)에서 허용 필요]

  <배포 도메인>                   사이트 본체 (Cloudflare      필수      접속 자체 불가
   (*.pages.dev 또는 커스텀 도메인) Pages 호스팅)
  rwplqifhmlduukipnksm            Supabase REST/RPC            필수      로그인·공지 등

7. 관리자 인증 보안 [완료]
----------------------------------------------------------------

[인증 방식]
- ID + 비밀번호 (Google 계정 불사용)
- 비밀번호 저장 : pgcrypto bcrypt 해싱 (crypt + gen_salt('bf'))
- 비교 방식 : crypt(입력값, stored_hash) = stored_hash (복호화 불가)

[로그인 실패 정책]
- 5회 연속 실패 → admin_status = 'locked', 30분 잠금
- 자동 해제 : admin_locked_until 경과 시
- 수동 해제 : 슈퍼관리자 DB 직접 처리 (UI 예정)
- 로그인 성공 시 fail_count 초기화

[세션 관리]
- 저장 방식 : localStorage (창 간 공유 — admin.html 새 창에서 자동 인식)
- 저장 항목 : adm_id, adm_name, adm_super
- 자동 로그아웃 : 5분 무활동 시 (마우스·키·클릭으로 타이머 초기화)
- 수동 로그아웃 : [로그아웃] 버튼 클릭 → localStorage 삭제 → 창 닫힘
- 네트워크 오류 : 세션 유지 (UNAUTHORIZED 응답 시에만 세션 삭제)

[API 키 / 보안 설정]
- Supabase RLS : admin_master, user_textnotice 모두 활성화
- 브라우저 노출 키 : publishable key (sb_publishable_…) — 신규 API 키 체계
    - 공개용 키, anon 역할로 동작 → RPC GRANT(anon) 적용받음
    - RLS + SECURITY DEFINER RPC로 보호 (테이블 직접 접근 없음)
- 레거시 키 (JWT 형식 anon / service_role) : 비활성화(disabled) 처리
    - 과거 service_role 키가 브라우저에 노출된 이력 → 레거시 키 폐기로 무력화
- secret key (sb_secret_…) : 브라우저 미사용 (서버 전용, 노출 금지)

12. 공지사항 화면 상세 [완료]
----------------------------------------------------------------

[일반 페이지 — 공지사항 카드]
- 데이터 : get_notices_public RPC (비공개 제외, 고정 우선 정렬)
- 고정 공지 : 상단 고정 목록 (고정 배지)
- 일반 공지 : 스크롤 목록 (최대 3행, 141px)
- 각 항목 클릭 → 페이지 내 모달(noticeDetailModal)로 상세 표시
  (이벤트 위임으로 목록 클릭 처리)
- 상세 모달 구성 :
  - 헤더 : '공지사항' + 닫기(✕)
  - 제목
  - 메타 라인 : [날짜] [게시자: 관리자 고정]  …  [조회 : N회] (우측 끝 정렬)
    · 게시자는 '관리자' 고정 표시 (이메일 미노출)
    · 조회수는 로드 시점 저장값 표시
  - 본문 (white-space: pre-wrap)
- 조회수 : 모달 오픈 시 increment_notice_readcnt RPC 호출
  (관리자 localStorage 키(adm_id) 있으면 증가 안 함)

[관리자 페이지 — 공지사항 CRUD]
- 목록 : get_notices_admin RPC (전체, 숨김 포함, 최대 10건)
  - 컬럼 : 제목 / 등록일 / 조회 / 고정(📌/—) / 게시(✅/🔒) / 삭제
  - 고정·게시 상태 : 읽기 전용 아이콘 표시 (편집 모달에서 변경)
- 등록/수정 모달 :
  - 입력 : 제목(필수) / 내용 / 작성자(신규 시 admin_id 자동 입력) /
            고정 체크박스 / 비공개 체크박스
  - 저장 : save_notice RPC (p_id=0 → INSERT, p_id>0 → UPDATE)
  - 삭제 : delete_notice RPC (confirm 후 실행)
- 날짜 : 등록 시 now() — GAS 동기화 형식과 동일
